## 系統介紹

FiftyOne Manager 是一個網頁管理工具，讓您能夠：

### 🎯 主要功能

- **管理多個 FiftyOne 實例** - 同時開啟多個資料集
- **瀏覽和檢視資料集** - 視覺化您的圖片和標註
- **編輯 YOLO 標籤** - 直接修正標註錯誤
- **智慧去重** - 自動偵測重複的圖片
- **即時監控** - 查看實例運行狀態

## 管理 FiftyOne 實例

### 1️⃣ 新增實例

點擊右上角的 **「+ 新增實例」** 按鈕。

![新增實例按鈕](images/05_add_instance_button.png)
> **截圖說明**：新增實例按鈕位置

#### 填寫實例資訊

![新增實例表單](images/06_add_instance_form.png)
> **截圖說明**：新增實例對話框

**必填欄位說明**：

| 欄位 | 說明 | 範例 |
|------|------|------|
| **實例名稱** | 給這個實例取一個容易識別的名字 | `骰子資料集_1106` |
| **連接埠** | 系統會自動建議可用的連接埠 | `5151` |
| **資料集路徑** | 選擇您的資料集所在位置 | `/data/datasets/dice/1106` |
| **類別檔案** | 選擇定義物件類別的檔案 | `classes.txt` |
| **IoU 閾值** | 去重檢測的相似度標準 | `0.8` |

#### 選擇資料集路徑

點擊 **「瀏覽」** 按鈕，會開啟資料夾瀏覽器：

![資料夾瀏覽器](images/07_folder_browser.png)
> **截圖說明**：瀏覽資料集資料夾

**資料集結構要求**：

您的資料集資料夾必須包含：
- `images/` 資料夾 - 存放所有圖片
- `labels/` 資料夾 - 存放對應的標註檔案

```
您的資料集/
├── images/
│   ├── img001.jpg
│   ├── img002.jpg
│   └── ...
└── labels/
    ├── img001.txt
    ├── img002.txt
    └── ...
```

![資料集結構](images/08_dataset_structure.png)
> **截圖說明**：正確的資料集資料夾結構

#### IoU 閾值設定

**IoU 閾值**決定系統如何判定兩張圖片是否重複：

- **0.9**：非常嚴格，只有幾乎相同的才算重複
- **0.8**（建議）：平衡的設定，適合大多數情況
- **0.7**：較寬鬆，更容易找出相似圖片

![IoU 說明](images/09_iou_explanation.png)
> **截圖說明**：IoU 閾值視覺化比較

#### 儲存實例

填寫完成後，點擊 **「儲存實例」** 按鈕。

系統會檢查：
- 實例名稱是否重複
- 連接埠是否可用
- 資料集路徑是否存在
- 資料夾結構是否正確

如果一切正常，新實例會出現在主畫面的實例列表中。

![實例已建立](images/10_instance_created.png)
> **截圖說明**：成功建立實例

1. 載入資料集

4. 啟動 FiftyOne 服務

![啟動中](images/12_starting_process.png)

等待幾秒鐘後，狀態會變成 🟢 **運行中**。

### 🔍 理解去重功能

系統會自動比較圖片的標註，找出重複的內容。

#### 去重如何運作

![IoU 計算](images/21_iou_calculation.png)
> **截圖說明**：IoU 相似度計算示意圖

系統會：
1. 比較兩張圖片的標註
2. 計算物件邊界框的重疊程度（IoU）
3. 如果所有物件都很相似，就標記為重複

#### 重複圖片的處理

重複的圖片會被移到 `duplicate/` 資料夾：

```
您的資料集/
├── images/          ← 保留的圖片
├── labels/
└── duplicate/       ← 重複的圖片
    ├── images/
    └── labels/
```

![去重結果](images/22_duplicate_folder.png)
> **截圖說明**：去重後的資料夾結構

### 資料集相關

#### Q14：YOLO 標籤格式是什麼？

**說明**：

YOLO 標籤檔案是純文字檔（.txt），每行代表一個物件：

```
<類別ID> <x中心> <y中心> <寬度> <高度>
```

**範例**：
```
0 0.5 0.5 0.3 0.4
2 0.7 0.3 0.2 0.2
```

**說明**：
- 類別 ID：0-6（對應不同的骰子點數）
- 所有座標值都是 0.0-1.0 之間的比例值
- 用空格分隔

![YOLO 格式](images/65_yolo_format.png)
> **截圖說明**：YOLO 標籤格式說明

---

#### Q15：去重後的圖片可以復原嗎？

**可以！**

重複的圖片只是被移到 `duplicate/` 資料夾，不會被刪除。

**復原方法**：

1. 開啟資料集資料夾
2. 找到 `duplicate/images/` 和 `duplicate/labels/`
3. 將需要的檔案移回 `images/` 和 `labels/`
4. 在 FiftyOne Manager 中重新啟動實例

![復原重複](images/66_restore_duplicates.png)
> **截圖說明**：手動復原重複圖片

---

#### Q16：如何知道有哪些類別？

**解決方法**：

檢查資料集資料夾中的 `classes.txt` 檔案：

```
one
two
three
four
five
six
invalid
```

每一行代表一個類別，順序對應類別 ID（從 0 開始）。

## 💡 使用技巧

### 提高工作效率

1. **使用鍵盤快速鍵**
   - 在 FiftyOne 中用 ← → 快速切換圖片
   - 在編輯器中用數字鍵快速切換類別
   - Ctrl+S 快速儲存

2. **善用篩選功能**
   - 先篩選出需要檢查的圖片
   - 集中處理特定類別
   - 使用標籤數量篩選找出可能的問題

3. **批次處理**
   - 一次開啟多個實例處理不同資料集
   - 使用多個瀏覽器分頁同時工作

4. **定期儲存**
   - 編輯器中經常按 Ctrl+S
   - 避免一次編輯太多再儲存

---

### 避免常見錯誤

❌ **不要**：
- 不要同時在 FiftyOne 和編輯器修改同一張圖片
- 不要忘記重新整理 FiftyOne（F5）
- 不要在實例運行時刪除資料夾
- 不要手動編輯 .txt 檔案時使用錯誤格式

✅ **建議**：
- 編輯完標籤後立即儲存
- 定期備份重要資料集
- 遇到問題先查看日誌
- 不確定時詢問系統管理員
